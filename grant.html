Grant a credential

Hand a secret to the runtime as a scoped grant. The value is NIP-44-encrypted to Nactor's key in your browser — it never leaves this page in the clear, and only Nactor's key can open it. It lives in Nactor's memory, revocable in one click, never on disk.

What happens. Your browser encrypts the value to Nactor's npub (NIP-44), then signs a NIP-98 request as you (the Director). Nactor decrypts it with its own key, holds it in RAM, and injects it only when your code brokers a call — the consumer never sees it. Rotate or revoke anytime; a redeploy that clears RAM simply means re-granting. Nothing here is written to disk.
